# 운용 모델 카드 — 저회전 추세 앙상블

> 현재 라이브(모의)로 운용 중인 전략의 출처·데이터·검증·한계를 한곳에 명시한다. (단일 출처)

## 식별
- **모델명**: 일봉 추세 앙상블 (ensemble)
- **확정·배포일**: **2026-06-20**
- **종류**: ML 학습 모델 아님 — **규칙 기반 전략을 walk-forward로 선별/검증**한 것(가중치 학습 없음)
- **코드**: `trading/strategy/ensemble.py` · `trading/strategy/trend_signal.py` (라이브: `trading/strategy/live_ensemble.py` → `trading/strategy/commander.py`)

## 전략 사양
- **타임프레임**: 일봉(daily), long-or-cash(공매도 없음)
- **구성**: 3속도 추세 앙상블 — 단기/장기 SMA **5/40 · 10/60 · 20/100** 의 목표비중 가중평균
- **사이징**: 변동성 타게팅(목표변동성/실현변동성, 상한 1.0)
- **재조정**: 합성 목표비중 드리프트가 밴드(`ENSEMBLE_REBALANCE_BAND=0.5`) 초과 시만(저회전)
- **운용 유니버스**: **BTC/ETH** (`ENSEMBLE_SYMBOLS`)

## 학습/검증에 사용한 데이터
- **소스**: 업비트 일봉 (REST `/v1/candles/days` → ClickHouse `candles_1d`)
- **기간**: **2019-11-25 ~ 2026-06-19** (종목당 **2,399봉**, ~6.6년)
- **종목**: KRW-BTC, KRW-ETH

## 검증 방법·결과
- **방법**: walk-forward(롤링 IS 180일 / OOS 90일), 고정 파라미터(per-fold 최적화 회피 — 과적합 방지),
  다중시도 보정 **Deflated Sharpe / PSR**
- **결과(BTC/ETH 6.6년 OOS)**: OOS 합성수익 **+1,594%**, 양수 fold **15/23(65%)**, 연율 Sharpe **~1.47**, **PSR 1.000**
- **교차검증**: BTC·ETH 개별로도 PSR ~1.0(풀링 우연 아님)
- 상세: [`docs/baseline.md`](baseline.md) §4

## 배포·운용
- **환경**: GCE **2-VM** — 상시 **틱 수집 VM** `coin-trader-vm`(e2-small, Kafka+CH+PG+틱 레코더만) + 온디맨드 매매 VM `coin-trade-vm`(로컬 DB 자기완결)
- **파이프라인(코인: 매일 01:00 UTC)**: Cloud Scheduler가 매매 VM 기동 → 로컬 DB 기동 + 크립토 일봉 REST 백필 → `trade_once`가 일봉 합성 목표비중 산출(현재가=업비트 REST) → 주문·체결 **동기 처리**(매매 VM 로컬 DB) → 자가 종료. (스트리밍 `commander`/`engine`은 로컬 개발용으로만 존재) 같은 VM이 부팅 시각 분기로 평일 15:00 KST엔 KR 주식, 평일 15:30 ET엔 US 주식도 실행 — 스케줄 정본은 `DEPLOY.md` §11 (주식 모델 경위는 `docs/ml_progress.md`).
- **거래 성격**: **모의(paper)** — 실거래 거래소 API 없음(가상 잔고). 초기 가상자본 ₩10,000,000
- **대시보드**: 온디맨드(매매 VM 상주) — 텔레그램 `/start_vm` 기동 후 SSH 터널 조회, `/stop_vm`로 종료. 상시 공개 URL 없음(비용 절감).

## 한계·주의 (정직)
- OOS 구간이 **2020-21·2023-25 대형 강세장**을 포함 → 추세추종에 유리했던 환경. 다른 레짐 일반화는 미검증.
- 단일종목 추세는 경계선(2년 표본에선 DSR<0.95였고, 6.6년으로 늘려야 충족) — **표본 길이에 민감**.
- 학습/검증 데이터는 **2026-06-19까지** — 이후 시장은 모델이 보지 못함(개념적 표류 가능).
- 검증은 일봉 종가 기준(슬리피지=0, 지연=0 이상화). 라이브 모의는 봉 종가 시뮬레이션 체결.
- `candles_1d`는 일일 최신화 필요(`aggregator/daily`) — 아니면 대시보드 스탠스가 마지막 적재일에 고정.
