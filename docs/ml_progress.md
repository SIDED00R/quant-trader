# 주식 ML 파이프라인 — 점진적 개선 연대기

> 각 단계: **무엇을 추가했나 → 왜 → 결과(측정)**. 모든 측정은 동일 규율(purged/embargo walk-forward CV + Rank IC/ICIR + Newey-West 겹침보정 t값 + 롱숏 Sharpe, 시장별 US/KR 분리). 권위 기록은 git 커밋(#143~), 큐레이트 표는 [ml_models_research.md](ml_models_research.md). **수치는 생존편향(현 구성종목 소급) 미보정이라 과대 — 절대값보다 *증분*과 *방향*을 본다.**

## 0. 토대 — 전체 유니버스 검증 (Issue#143 / PR#144)
- **무엇**: KOSPI200+KOSDAQ150+S&P500+NASDAQ100 구성종목 856종목 일봉 7.1년 백필 → 횡단면 모멘텀 walk-forward.
- **왜**: 6·30종목 소표본에선 −33~+108% 요동으로 "기각"처럼 보였음. breadth(종목 폭)가 빠진 조각인지 확인 필요.
- **결과**: 사전확정 lb60/N10 = KR +481%/DSR0.990, US +604%/DSR0.998. 룩백/N 전반 안정(과적합 아님), 슬리피지 20bps 생존. **breadth가 결론을 뒤집음.** 단 생존편향이 핵심 미결.

## 1. ML 피처 토대 (Issue#145)
- **무엇**: OHLCV 파생 피처 ~58개(모멘텀·변동성 레인지추정·유동성·Alpha158 KBar/롤링·기술지표) + Rank IC/ICIR/NW-t 유용성 하니스 + purged/embargo CV.
- **왜**: 논문 전수조사(213피처) 결론 — GKX 최상위 예측변수 대부분이 OHLCV로 계산 가능. 모델보다 피처가 90%.
- **결과(단변량 IC, 참고)**: Amihud 유동성이 양 시장 최강(US/KR NW_t~3.1). KR=교과서 이상현상(저변동성·복권·단기반전), US=강세장/생존편향 왜곡. → KR/US 분리 확정.

## 2. GBDT 베이스라인 (게이트)
- **무엇**: LightGBM, 시드앙상블, 라벨=횡단면 z-score.
- **회귀 → lambdarank 손실 교체**: 평가(Rank IC)와 학습목표 정렬.
- **결과(US, OOF 5시드6fold)**: 회귀 2.06%/NW_t1.6 → **lambdarank 3.09%/NW_t1.8**. KR은 0.99%/0.67%로 약함(OHLCV만으론 OOS 신호 부족 — KR은 외국인수급 필요 시사).

## 3. 데이터-우선 전환 + 외부데이터 ablation (핵심)
- **무엇/왜**: "모델 검증과 무관하게 데이터를 다 수집·저장"(재사용 자산·PIT 소실 방지). `batch/data/` 멱등 수집기로 영구 저장: SEC EDGAR 펀더멘털·FRED 매크로·US PIT멤버십·13F 기관보유·SEC SIC 섹터. **각 외부데이터를 ablation으로 유용성 측정 후 선별.**
- **결과(US lambdarank, 누적)**:
  | 추가 | 이유 | Rank IC | NW_t | 판정 |
  |---|---|---|---|---|
  | OHLCV 전용 | 베이스 | 3.09% | 1.8 | 기준 |
  | **+ 펀더멘털**(EDGAR 공시) | GKX 가치/규모/수익성 팩터 | **3.41%** | **2.1** | 채택(유의성 첫 돌파) |
  | **+ 13F**(기관 보유·QoQ) | 기관 누적/분산 신호 | **3.56%** | **2.2** | 채택 |
  | **+ 섹터**(산업모멘텀 indmom) | GKX top-7, 섹터중립 | **3.78%** | **2.3** | 채택(현 챔피언) |
  | + 매크로(금리·VIX·환율) | 레짐 | 2.05% | 1.5 | **제외(시장수준→과적합)** |
- **교훈**: **검증된 외부데이터(펀더멘털·13F·섹터)는 누적 개선**(NW_t 1.8→2.3). **매크로(시장수준)는 무지성 추가 시 과적합** → ablation 선별 필수.
- **KR**: 외국인수급·공매도·KR펀더멘털(KRX/DART 키 대기) 추가 시 동일 검증 예정.

## 4. DL 비교 (완료, Issue#148) — 트리 압승, DL 미달
- **무엇**: GBDT(lambdarank) vs MLP vs GRU(raw OHLCV 5채널×60일) head-to-head. 동일 데이터·purged CV·자체완결 스크립트(`batch/ml/colab_train.py`)로 같은 파이프라인 비교.
- **왜**: 시퀀스 DL이 트리(공학피처)와 직교 정보원일 수 있나? 단 tabular는 GBDT가 강하다는 prior.
- **결과(최종, 동일 파이프라인)**:
  | 모델 | 입력 | Rank IC | NW_t | LS_Sharpe | 판정 |
  |---|---|---|---|---|---|
  | **GBDT**(lambdarank) | 공학피처 70 | **3.40%** | 2.2 | 1.20 | 챔피언 |
  | MLP | 같은 70 | 1.22% | 1.5 | 0.15 | 미달 |
  | GRU | raw OHLCV 5ch×60일 | 0.27% | 0.3 | 0.38 | 사실상 0 |
- **인프라**: 무료 Colab GPU는 한도소진·세션회수로 실패 → **Kaggle 커밋세션(P100)** 으로 GRU 풀학습(5시드×6fold×40ep, 112분). P100=sm_60이라 기본 torch(cu128) 미지원 → 커널서 `torch==2.4.1+cu118` 재설치.
- **교훈**: GRU는 **GPU 풀학습으로도 엣지 없음**(축소 CPU 0.37%와 일치 → 과소적합 아닌 **구조적 미달**). 정형 횡단면 금융데이터는 GBDT가 DL 압도(문헌 일치). GRU는 입력 핸디캡(펀더/13F 없이 raw만)도 일부. **DL 보류 — 다음 개선은 피처·레짐·생존편향 통제 방향.**

## 5. GBDT 하이퍼파라미터 튜닝 (누설 없음, Issue#148)
- **무엇/왜**: 고정 HP가 최적인지 검증. **시간분리**(앞 45% 날짜=튜닝창 탐색 / 뒤 55%=평가창은 탐색 미관여)로 누설 차단.
- **결과**: 24조합 탐색 → held-out 평가창서 default **6.62%** → tuned **6.82%**(소폭). best = 더 **보수적**(num_leaves15·min_child300·feature_fraction0.6) → 기존 고정값이 이미 거의 최적. 대규모 탐색은 효용 낮음(이득 작고 다중검정 과적합 위험).
- **부수 발견**: 평가창(최근 55%) IC가 전체기간(3.40%)보다 훨씬 높음 → 엣지가 **최근 구간에 집중**(레짐효과·생존편향 주의).
